# 项目变更摘要

- 用途：在发布或里程碑需要上下文时保存面向人的简短摘要。
- 边界：允许用户可见/工程变化、迁移、验证和风险；不替代 Git、commit message、AI timeline 或 release notes。
- 读取时机：发布检查、版本回顾或新人了解里程碑时读取。
- 更新与责任：发布负责人或实现者在里程碑评审时更新，普通小修复可只保留提交/任务报告。
- 权威来源：代码历史以 Git 为准，验证以真实命令和平台记录为准；发布流程见 [`../deployment/README.md`](../deployment/README.md)。
- 项目示例：`2026-07-18：建立 AI 基础设施和前端质量卡口；H5 已检查，微信平台验收待完成。`

建议记录日期/版本、变化、迁移、验证、已知风险和负责人，不把未运行检查写成完成。
