# 需求资料

本目录保存已评审的需求、范围和验收口径，不保存临时聊天、实现规则或 AI 推测。产品/任务负责人提供和维护；开发者与 AI 在实现相关功能前只读取对应需求。

每份需求至少包含：背景与来源、目标用户、目标、范围、非目标、主/异常流程、角色权限、验收标准、相关设计/API、依赖、负责人和待确认项。建议按功能命名，例如 `profile-page.md`。

当前状态：仓库没有真实 PRD。原因是现有首页/登录页属于初始化或脚手架，不能代表业务；产品负责人最晚在首个真实业务功能开始前提供已评审资料。确认后新增具体文件，并同步 [`../../ai/knowledge/business.md`](../../ai/knowledge/business.md) 和 glossary。AI 可以整理草稿，不能自行决定产品范围。
