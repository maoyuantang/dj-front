# 审查报告

- 用途：保存需要跨会话跟踪的代码、架构或安全审查报告。
- 边界：允许有范围、findings、证据和生命周期的报告；不放日常聊天或自动检查原始输出。
- 读取时机：Review 任务和后续问题关闭复核时读取对应报告。
- 更新与责任：Reviewer 创建/更新，问题负责人补状态；结论失去长期价值后归档或删除。
- 权威来源：审查方法见 [`../prompts/review-code.md`](../prompts/review-code.md)，实际结论必须指向目标 diff/文件行和测试证据。
- 项目示例：命名为 `2026-07-18-auth-review.md`，按严重度记录鉴权问题；本次初始化不制造虚假报告。

报告至少包含范围、版本/diff、findings、测试缺口、负责人、结论和状态。
