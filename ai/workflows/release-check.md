# 发布检查工作流

- 触发条件：人明确要求检查某个平台/环境的发布准备，不代表授权上传或发布。
- 输入：目标平台、环境来源、版本/变更范围、发布负责人、验收和回滚方式。
- 前置检查：工作区、真实 package scripts、manifest/pages/env、敏感信息、OpenAPI/部署资料和已知风险。
- 阶段：type-check -> lint/style/format -> unit test -> 目标 build -> 开发者工具/真机/网络人工验收 -> 发布说明和回滚证据。
- 质量门：只使用真实命令；H5、微信 build、开发者工具、真机和生产环境分别记录，负责人签字/确认可追溯。
- 失败处理：任何门失败则停止后续发布动作，保留精确错误、影响、修复负责人和重试条件。
- 完成产物：检查矩阵、构建产物状态、人工验收、敏感信息检查、风险、回滚和 signoff；未经授权不上传。
