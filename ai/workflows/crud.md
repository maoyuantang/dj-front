# CRUD 工作流

- 触发条件：资源的 list/detail/create/update/delete operation、权限和页面需求已确认。
- 输入：OpenAPI、字段/类型、分页筛选、权限、错误语义、页面设计和验收。
- 前置检查：定位全部相关 operation，检查 request/store/表单/列表既有模式和路由归属。
- 阶段：契约与类型 -> API 映射 -> 列表分页/筛选 -> 详情/表单/校验 -> 防重提交/删除确认 -> 状态同步 -> 错误/权限 -> 验证。
- 质量门：不猜字段，编辑回填一致，删除/更新后状态一致，loading/empty/error/401/403 完整，相关检查和联调证据有效。
- 失败处理：契约、权限或破坏性操作语义缺失时停止对应部分，保留可独立验证的工作并列负责人问题。
- 完成产物：operation 映射、页面/API/store/类型、状态转换、验证、待确认和风险。
