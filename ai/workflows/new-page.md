# 新页面工作流

- 触发条件：已有确认的页面目标和实现授权。
- 输入：PRD/验收、设计版本、入口/返回、目标平台、API operation、权限和页面状态。
- 前置检查：工作区、`src/pages.json`、现有页面/组件、request/store 边界、真实 scripts。
- 阶段：确认状态与路由 -> 决定主包/真实业务分包 -> 设计页面/组件边界 -> 注册页面 -> 接入数据/权限 -> 实现生命周期和平台差异 -> 分层验证。
- 质量门：参数/导航正确，loading/empty/error/disabled 完整，适用的 type/lint/style/test/build 通过，设计与平台人工证据单列。
- 失败处理：在失败阶段停止，保留用户改动，说明错误、当前 diff、恢复方式和未完成项；不创建假 API 或空分包绕过阻塞。
- 完成产物：页面路径、相关组件/API/store、路由注册、检查结果、平台未验证项、文档更新和风险。
