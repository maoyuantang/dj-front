# UI 与交互规则

样式实现以已确认 PRD、设计稿、设计 Token 和素材来源为准；缺少设计时明确假设并获得人工评审，不擅自把 uview-plus 默认样式当成项目品牌。页面必须处理 loading、empty、error、disabled、success、retry 和重复操作状态。

关注小屏、长文本换行、安全区、键盘、触摸目标、图片比例、可访问标签和平台差异。固定格式控件使用稳定尺寸/比例，动态内容不能导致布局跳动或遮挡；条件编译保持最小范围。素材需要来源和授权，不声称未配置的 Figma/MasterGo/MCP 已可用。

目标为微信小程序时，H5 浏览器只能辅助检查；关键流程需在微信开发者工具和必要真机尺寸上记录截图/交互结果。正例：根据设计版本检查多状态和安全区。反例：只看桌面 H5 截图就宣称小程序视觉验收通过。
