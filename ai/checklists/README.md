# 验收清单

- 用途：在完成任务或 Review 前提示需要的证据。
- 边界：允许 DoD、UI 和 API 检查项；不替代测试、设计评审或平台验收。
- 读取时机：实现完成、自审或交付前选择相关清单；不需要每个任务加载全部清单。
- 更新与责任：质量门或项目验证能力改变时，由实际变更者更新。
- 权威来源：真实命令来自 `package.json` 和 [`../rules/testing.md`](../rules/testing.md)，专项约束来自对应 rules。
- 项目示例：H5 截图不能勾选“微信真机已验证”，两项必须分开记录。

每个勾选项必须有真实证据；未执行的保留未勾选并说明原因。
