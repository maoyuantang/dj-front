# 长期记忆

- 用途：保存跨任务仍有效的已确认决定和重要项目事件。
- 边界：允许 decisions/timeline；不作为聊天归档、任务 TODO、Git changelog 或技术事实副本。
- 读取时机：任务涉及历史架构约定或治理里程碑时选择性读取。
- 更新与责任：决定被接受/替代时由参与者更新；重要治理事件由执行者记录。
- 权威来源：决定格式参考 [`../templates/decision-record.md`](../templates/decision-record.md)，代码现状仍看 [`../knowledge/project.md`](../knowledge/project.md)。
- 项目示例：统一规则源是长期决定；某次 lint 失败只写任务报告，不写 memory。

被替代的决定保留历史并链接新决定，普通功能修改无需逐条写 timeline。
