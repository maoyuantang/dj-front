# 重构 Prompt

- 目标：在 `{{scope}}` 内因 `{{reason}}` 重构，外部行为保持不变。
- 必要输入：当前行为、必须保持的不变量、文件边界、调用方、验收和基线证据。
- 开始前读取：bootstrap、global/frontend/architecture/testing 规则、全部直接调用方和相关测试。
- 约束：不改变行为、API、依赖或无关格式；超出范围先报告。
- 执行步骤：记录基线 -> 识别最小结构问题 -> 分小步修改 -> 每步比较输出/调用方 -> 回归检查。
- 验收标准：公开契约和用户行为不变，重复或复杂度确实降低，相关检查通过。
- 输出格式：不变量、修改文件、前后对比、验证和剩余风险。

示例：只提取两个页面已重复的字段映射，不顺手改页面视觉和请求库。
