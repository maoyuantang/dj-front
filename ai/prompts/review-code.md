# 代码审查 Prompt

- 目标：审查 `{{scope}}`，默认不修改代码。
- 必要输入：diff/分支/文件范围、需求与验收、目标平台、已执行检查。
- 开始前读取：bootstrap、global/testing/security、相关 architecture/API/UI 规则、目标 diff、调用方和测试。
- 约束：问题优先，按严重度排序；每条给文件/行、触发条件、影响和修复方向；不把风格偏好当 Bug。
- 执行步骤：核对需求 -> 读 diff 与上下游 -> 检查回归/安全/平台 -> 评估测试覆盖 -> 复核证据。
- 验收标准：每条发现可复现或有明确逻辑证据；没有发现时直说，并列测试缺口/残余风险。
- 输出格式：findings、假设/问题、测试缺口、简短总结。

示例：指出 401 分支会重复跳转时说明调用路径和用户影响，而不是只写“建议优化错误处理”。
