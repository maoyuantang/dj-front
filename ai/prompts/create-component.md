# 创建组件 Prompt

- 目标：创建或扩展组件 `{{component_name}}`。
- 必要输入：两个真实使用场景或稳定交互边界、props/events/slots、设计来源、状态和适配范围。
- 开始前读取：bootstrap、global/frontend/ui/architecture 规则、现有组件、两个调用方及相关测试。
- 约束：展示组件不直接调用业务 API；契约有类型；尺寸和动态内容稳定；不为了“以后可能用”抽象。
- 执行步骤：搜索重复能力 -> 定义契约和状态 -> 实现 -> 接入调用方 -> 验证交互、文本和平台表现。
- 验收标准：复用理由成立，契约清晰，无隐藏业务耦合，相关检查通过。
- 输出格式：组件契约、调用位置、修改文件、验证、未验证和风险。

示例：两个页面都需要相同空状态时抽取 `AppEmptyState`；只有一个页面的局部标题不要提前做通用组件。
