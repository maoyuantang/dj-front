# 创建页面 Prompt

- 目标：实现 `{{page_name}}`。
- 必要输入：PRD/用户目标、入口与返回路径、目标平台、设计来源、相关 API operation、验收标准。
- 开始前读取：bootstrap、project、global/frontend/ui/architecture 规则、new-page workflow、目标路由和相关资料。
- 约束：只注册已确认路径；确认主包/分包和导航 API；复用现有 request/env/store；不猜接口或视觉。
- 执行步骤：核实输入与页面状态 -> 检查现有页面/组件 -> 设计边界 -> 注册路由 -> 实现数据和交互 -> 处理生命周期/平台差异 -> 分层验证。
- 验收标准：页面可达，参数和返回明确，loading/empty/error/disabled 完整，适用的类型/质量/测试/构建通过，平台人工项已报告。
- 输出格式：读取和修改文件、路由/组件边界、实际检查、未验证平台项、风险与后续。

示例：创建商品详情页，参数为 `id`；若没有已验证商品 operation，只实现明确的空/错误状态和类型边界，不编造响应字段。
