# 修复 Bug Prompt

- 目标：诊断并修复 `{{bug}}`。
- 必要输入：复现步骤、实际/预期结果、目标平台、日志/截图、影响范围和修复授权。
- 开始前读取：bootstrap、global、bugfix workflow、目标代码/调用方/最近测试，再按根因读取 frontend/API/security/testing 规则。
- 约束：先复现和证明根因；仅诊断请求不改代码；不借机重构或隐藏失败。
- 执行步骤：稳定复现 -> 收集证据 -> 缩小边界 -> 说明根因和最小修复 -> 获授权后实现 -> 聚焦及回归检查。
- 验收标准：同一复现修复前失败、修复后通过；相关回归通过；平台未测和残余风险明确。
- 输出格式：复现证据、根因、补丁范围、验证对比、未验证和残余风险。

示例：微信端返回页面重复请求，先证明监听或生命周期重复注册，再只修复对应注册/清理逻辑。
